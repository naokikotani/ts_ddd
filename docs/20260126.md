アーキテクチャ設計

レイヤードアーキテクチャ
→ システムを階層状に積み上げる。
シンプルで分かりやすいが、下層の変更（DB等）がビジネスロジックに影響を与えやすい

ヘキサゴナルアーキテクチャ
→ アプリケーションの中心部分と周辺部分を明確に区別する。
外部とのやりとりはポートというインターフェースを介して行う

クリーンアーキテクチャ
→ ビジネスルールを技術的な詳細から独立させる

オニオンアーキテクチャ
→ 中心にドメインモデル、その外側にアプリケーションサービス層
最も外側にインフラ層（DBアクセスやUIなど）
常に外側から内側に依存し、中心のドメインモデルはどこにも依存しない

下三つは基本的に同じ思想（ビジネスロジックを外部の詳細から分離する）で
表現方法が異なるだけという理解をした
