戦術的設計を行わないとどうなるか
1. データ中心の実装
ビジネスロジックがデータの操作に埋もれる

2. 一貫性のない実装
ここの機能は動作するものの、コードとしての一貫性が失われている

3. ドメインの知識が埋もれる
ビジネスの本質的な知識がコードから読み取りにくくなる

4. 変更に弱い実装
機能の追加や変更が困難になる

戦術的設計を取り入れることでこう改善される
1. ドメインモデル中心の実装
ビジネスの本質をコードで表現する

2. 一貫性のある実装
戦術的設計パターンを活用した統一的な実装

3. ドメイン知識を明示的に表現
ビジネスの本質的な価値をコードで表現する

4. ビジネス中心の変更に強い実装
将来の変更や拡張を考慮した設計
